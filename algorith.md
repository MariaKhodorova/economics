# Алгоритм выбора организационно-правовой формы

## Логика принятия решений

### Основной алгоритм (реализован в коде):

```javascript
function getOPFRecommendation(activity, scale, founders, investment, responsibility) {
    if (founders === 1 && scale === 'small' && responsibility === 'full' && investment === 'no') {
        return opfOptions.ip; // Индивидуальный предприниматель
    } else if (founders <= 50 && investment !== 'yes' && (scale === 'small' || scale === 'medium')) {
        return opfOptions.ooo; // Общество с ограниченной ответственностью
    } else {
        return opfOptions.ao; // Акционерное общество
    }
}
```

## Детальное описание алгоритма

### 1. **Критерии для ИП (Индивидуальный предприниматель)**
```
ЕСЛИ:
    Количество учредителей = 1
    И Масштаб бизнеса = "малый"
    И Ответственность = "полная"
    И Инвестиции = "не требуется"
ТО:
    Рекомендовать ИП
```

**Обоснование**: ИП идеально подходит для solo-предпринимателей, готовых нести полную ответственность, без планов масштабирования и привлечения инвестиций.

### 2. **Критерии для ООО (Общество с ограниченной ответственностью)**
```
ЕСЛИ:
    Количество учредителей ≤ 50
    И Инвестиции ≠ "требуется сейчас"
    И (Масштаб = "малый" ИЛИ "средний")
ТО:
    Рекомендовать ООО
```

**Обоснование**: ООО - универсальная форма для большинства бизнесов с несколькими учредителями, где не требуется немедленное привлечение крупных инвестиций.

### 3. **Критерии для АО (Акционерное общество)**
```
ВО ВСЕХ ОСТАЛЬНЫХ СЛУЧАЯХ:
    Рекомендовать АО
```

**Обоснование**: АО рекомендуется для сложных случаев - крупный бизнес, необходимость инвестиций, много учредителей.

## Весовые коэффициенты параметров

### Приоритет параметров (от высокого к низкому):

1. **Количество учредителей** - самый строгий критерий
2. **Потребность в инвестициях** - ключевой для выбора между ООО и АО
3. **Масштаб бизнеса** - определяет потенциал роста
4. **Уровень ответственности** - важный, но гибкий параметр
5. **Вид деятельности** - в текущей реализации не влияет на выбор (резерв для расширения)

## Бизнес-логика принятия решений

### Дерево решений:

```
Начало
    ↓
Количество учредителей?
    ↓
Если 1 → Проверяем масштаб, ответственность, инвестиции
    ↓
Если подходит → ИП
    ↓
Если не подходит или учредителей >1 → Проверяем количество (≤50?)
    ↓
Если да → Проверяем инвестиции и масштаб
    ↓
Если подходит → ООО
    ↓
Если не подходит → АО
```

## Преимущества используемого алгоритма

### 1. **Простота и прозрачность**
- Четкие правила принятия решений
- Легко модифицировать и расширять
- Понятная логика для пользователя

### 2. **Соответствие законодательству**
- Учитывает ограничения по количеству участников
- Соответствует требованиям к уставному капиталу
- Учитывает уровни ответственности

### 3. **Масштабируемость**
- Легко добавить новые ОПФ
- Можно ввести весовые коэффициенты
- Возможность учета дополнительных параметров

## Примеры работы алгоритма

### Пример 1: Малый бизнес
```
Параметры:
- Учредители: 1
- Масштаб: малый  
- Инвестиции: не требуется
- Ответственность: полная
→ Результат: ИП
```

### Пример 2: Средний бизнес с партнерами
```
Параметры:
- Учредители: 3
- Масштаб: средний
- Инвестиции: возможно в будущем
- Ответственность: ограниченная
→ Результат: ООО
```

### Пример 3: Крупный проект
```
Параметры:
- Учредители: 15
- Масштаб: крупный
- Инвестиции: требуются сейчас
- Ответственность: ограниченная
→ Результат: АО
```
